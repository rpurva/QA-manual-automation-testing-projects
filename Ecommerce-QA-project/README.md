# E-commerce Manual Testing Project 

## Overview
This project demonstrates manual testing of an e-commerce application (Flipkart) for learning purposes. It focuses on validating key user workflows such as login, search, product selection, cart operations, and checkout process.

## Application Under Test
Flipkart (E-commerce Website) -> Used for practice and learning manual testing concepts.

## Test Artifacts
All test artifacts are maintained in a single Excel workbook with multiple sheets:
- Test Cases
- Bug Reports
- Test Summary

## Modules Tested
- Login Functionality
- Search Functionality
- Product Page
- Cart Functionality
- Checkout Process

## Testing Types Performed
- Functional Testing
- UI Testing
- Positive Testing
- Negative Testing

## Tools Used
- Excel (Test Case Design & Execution)
- Chrome Browser
- Manual Testing Techniques

## Sample Test Scenarios
- Validate login with valid and invalid credentials
- Verify search functionality with different inputs
- Add and remove products from cart
- Validate price updates in cart
- Checkout validation with empty address or invalid data

## Outcome
The application was tested for major user flows. Most functionalities worked as expected with proper validations in place.

## Conclusion
This project demonstrates my understanding of manual testing concepts including test case design, execution, defect reporting, and real-world e-commerce workflow validation.
